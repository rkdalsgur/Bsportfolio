연결 자료구조와 연결 리스트

연결 리스트의 노드: 원소를 표현하기 위한 단위 구조
(원소,주소)의 구조
연결 리스트의 경우에는 데이터 주소와 다음 주소를 기억하는 링크 필드로 구성됨
연결 구조는 논리 순서와 물리 순서를 굳이 꼭 일치시키지 않아도 괜찮다
연결 구조의 경우 데이터값과 꼬릿값에 뒤에 올 데이터의 주소값이 함께 존재하기 때문에,
순서대로 배치시키기가 쉽다.
리스트 이름: 연결 리스트의 시작을 가리키는 포인터 변수.
마지막 노드의 꼬릿값은 Null로 지정해서 리스트의 끝을 알림
*.data <- 연결 구조의 데이터값을 가르키는 명령어 ex) 월
*.link <-연결 구조의 주소값을 가르키는 명령어 ex) 150
단순 연결 리스트의 개념
노드가 하나의 링크 필드에 의해서 다음 노드와 연결되는 구조
연결 리스트에서의 데이터 삽입 방법
1. 삽입할 노드 준비: 공백 노드를 가져와 포인터 변수new가 가르키게 함
2. 새 노드의 데이터 필드에 값을 지정
3. 새 노드의 링크값 지정
4. 리스트의 앞 노드에 새 노드를 연결
연결 리스트에서의 데이터 삭제 방법
1. 앞 노드 찾기
2. 앞 노드에 삭제할 노드의 링크 필드 값 저장
3. 삭제한 노드의 앞뒤 노드 연결
대신 뒤에 참조값 때문에 잔여 데이터가 남아 메모리를 잡아먹음
